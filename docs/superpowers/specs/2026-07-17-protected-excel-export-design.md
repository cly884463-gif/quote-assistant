# 受保护 Excel 报价导出设计

## 目标

在现有“导出报价图片”功能旁增加“导出Excel表格”，生成可用 Excel 打开的 `.xlsx` 报价文件。文件完整保留报价内容、浅灰品牌水印，并通过工作表密码保护阻止日常编辑。

## 用户界面

- 在最终报价清单顶部操作区增加按钮“导出Excel表格”。
- 按钮与“导出报价图片”并列，桌面端和手机端均不得溢出或遮挡筛选项。
- 普通浏览器点击后直接下载 `.xlsx` 文件。
- 微信内置浏览器无法可靠下载 Blob 文件时，显示明确提示，引导用户使用右上角菜单在系统浏览器打开当前网页后导出。

## Excel 内容

- 第一行保留物流方式和送货方式。
- 明细表保留系列号、材料类别、产品名称、产品规格、施工次数、单桶涂布量、单位、数量、对应报价单价和对应价格合计。
- 经销商报价使用“经销商单价”“经销商合计”；渠道合作报价使用“渠道合作单价”“渠道价合计”。
- 明细后保留不含税合计、发票税金、含税报价和用户填写的备注。
- 表格底部保留完整“注意事项”。
- 金额单元格使用人民币数字格式，表头、边框、黄色顶部行和绿色价格列延续当前报价图片样式。

## 不可编辑保护

- 所有单元格保持锁定，并调用 ExcelJS 工作表保护功能。
- 保护密码作为实现常量使用，不在界面显示。
- 禁止插入、删除、格式化和修改单元格，允许用户正常打开和查看。
- Excel 工作表保护用于防止日常误改，不等同于文件加密；具备专业工具的用户仍可能解除保护。

## 水印

- 水印文字固定为 `新材联·New Material Union`。
- 使用浏览器 Canvas 生成透明 PNG 水印图，再通过 ExcelJS 嵌入工作表。
- 水印使用浅灰色、斜向重复排列，视觉强度与当前报价图片水印接近，不影响文字和金额阅读。
- 水印覆盖报价表主要区域，并随 Excel 文件保存和打印。

## 技术方案

- 使用本地 ExcelJS `4.4.0` 浏览器构建文件，不引用外部 CDN。
- 将浏览器可用的压缩构建文件放入 `web/vendor/` 并由 `web/index.html` 在 `app.js` 前加载。
- `web/app.js` 负责把现有 `calculateQuote()` 结果转换为工作表、应用样式、保护工作表、嵌入水印并触发下载。
- 不修改产品数据、报价计算、图片导出和 Netlify 部署模型。

## 测试与验收

- 静态测试确认 ExcelJS 本地脚本、导出按钮、事件绑定、工作表保护、水印文字和浏览器下载逻辑存在。
- 增加独立 Excel 导出数据模型测试，确认两种报价类型列名、合计、备注和注意事项完整。
- 完整运行现有测试并重新生成 Netlify 发布包。
- 验收标准：桌面浏览器可下载 `.xlsx`；Excel 打开后报价内容与当前清单一致、工作表不可编辑、水印清晰但不影响阅读。
